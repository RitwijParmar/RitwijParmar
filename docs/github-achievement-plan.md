## Weekly Routine

1. Open one small PR that improves a real project: tests, README clarity, benchmark notes, bug fixes, or deployment docs.
2. Merge through GitHub so the pull request contribution is recorded cleanly.
3. Add one issue or discussion answer where it helps another developer.
4. Keep fork work tied to upstream PRs; commits that stay only in forks are weaker profile signal.
5. Review the profile after GitHub refreshes achievements and contribution cards.
